# Skill: 多篇文献综述

## Goal
对多篇文献进行综述式分析，输出的重点是：
1. 该方向真正试图解决的底层问题是什么；
2. 不同文献提出了哪些核心 Idea；
3. 这些 Idea 如何演化；
4. 多篇文献共同支持了哪些科学共识；
5. 哪些分歧最值得关注，未来突破可能藏在哪里。

本 skill 的目标是构建“研究图谱”，而不是生成文献清单。

## Use this skill when
满足以下任一情况时启用：
- 用户要求对某一方向做文献综述；
- 用户要求比较多篇论文；
- 用户要求梳理某主题的发展脉络；
- 用户准备综述报告、开题汇报、专题读书会；
- 用户希望从一组文献里提炼共识、争议和潜在选题。

## Do not use this skill when
以下情况不优先使用本 skill：
- 用户只给了 1 篇论文并要求深读；
- 用户只要逐篇摘要；
- 用户只做翻译、摘录、表格化信息整理；
- 用户要评价单篇论文质量。

## Required inputs
尽量提取或整理以下内容：
- 研究主题 / 子方向
- 文献集合
- 时间范围（如有）
- 研究对象或体系范围
- 各篇工作的核心策略
- 关键证据、关键结论、主要冲突点
- 用户的应用场景：综述写作、组会、选题、开题等

如果用户给的文献较少，也要尽量建立“共识—分歧—演化”结构。

## Core rules
1. 不按“年份 + 作者 + 性能数字”写流水账。
2. 弱化个体，强化逻辑。
3. 优先按问题、策略、机制、阶段分组，不优先按年份分组。
4. 综述的核心产物是：主线、支线、共识、分歧、演化。
5. 具体材料、具体表征、局部性能，只有在影响主线判断时才重点写。
6. 对不同论文的比较，先看测试口径和边界条件，再看数字高低。
7. 少数反证文献必须单独标出，它们常常更有价值。
8. 综述输出要让读者看到“为什么这个方向会走到今天这一步”。

## Review framework
每次综述都优先回答以下五个问题：
1. 这个方向的底层问题是什么？
2. 学界围绕这个问题提出了哪些路线？
3. 每条路线的核心 Idea 是什么？
4. 哪些结论已形成共识？
5. 哪些冲突仍未解决？

## Grouping priority
优先按以下顺序组织文献：
1. 按核心科学问题；
2. 按策略类型；
3. 按机制假设；
4. 按研究阶段：
   - 概念提出
   - 扩展优化
   - 机制澄清
   - 应用推进

## Output requirements
输出必须严格使用以下结构与标题。

### 一、主题定义
- 综述主题：
- 核心科学问题：
- 本次纳入文献的范围与标准：

### 二、研究路线总览
用 3–6 条概括主要路线，每条路线一句话：
- 路线 A：
- 路线 B：
- 路线 C：

要求：
- 每条路线写“在解决什么 + 依赖什么逻辑”；
- 不写成作者名单。

### 三、Idea 演化逻辑
按路线或阶段逐段输出：

#### 路线 / 阶段名称
- 初始出发点：
- 核心 Idea：
- 代表性推进：
- 相比上一阶段真正前进了什么：
- 仍未解决的问题：

要求：
- 重点写“推进的本质”；
- 只在必要时提关键文献，不堆作者与年份；
- 区分“范式推进”和“参数优化”。

### 四、科学共识
列出 2–5 条最重要的共识：
- 共识内容：
- 共同证据基础：
- 它对后续研究意味着什么：

要求：
- 共识必须来自多篇文献的交叉支持；
- 不能把单篇结论误写成领域共识。

### 五、科学分歧
列出最值得关注的分歧：
- 分歧点：
- 观点 A 的依据：
- 观点 B 的依据：
- 造成分歧的可能原因：
- 这对未来研究的启发：

要求：
- 分歧可以来自机制解释、实验条件、材料体系、评价口径；
- 不允许把“结果不同”草率等同于“观点冲突”。

### 六、你的综合判断
必须回答：
- 当前最可靠的主线是什么？
- 哪些工作看上去不同，本质上却共享同一逻辑？
- 哪些方向表面热门，但证据还不够硬？
- 下一步最值得追的问题在哪里？

### 七、一段式综述摘要
用一段话总结整个方向，要求：
- 不写流水账；
- 不靠作者和年份串联；
- 直接写底层问题、主要路线、当前共识、关键分歧与未来切口。

## Optional add-ons
按用户需求追加：

### A. 10 分钟组会提纲
输出：
- 开场问题
- 三条主线
- 两条共识
- 一条最大分歧
- 结尾判断

### B. 综述前言风格改写
把上面的分析重写成正式综述文风，适合作为 review（综述）或 introduction（引言）初稿。

### C. 选题模式
在结尾增加：
- 潜在研究切口
- 可验证假设
- 需要补的关键证据
- 哪类反证最值得做

## Anti-patterns
禁止出现以下输出方式：
- “2020 年某某做了……2021 年某某做了……2022 年某某又做了……”
- “A 体系 10%，B 体系 12%，C 体系 15%”式的数字堆叠
- 把每篇文献各写一段，然后简单拼起来
- 只写共识，不写冲突
- 只写结果，不解释这些结果之间的继承、修正和分裂关系

## Style constraints
- 站位高于单篇解读；
- 强调抽象、比较、归纳、判断；
- 少写局部细节，多写逻辑结构；
- 默认中文输出；
- 语言紧凑，不堆术语，不做无意义赞美。

## Failure handling
遇到以下情况时这样处理：
- 文献数量少：仍然尝试提炼共识与分歧，但明确说明样本有限；
- 文献类型混杂：先分层，再判断是否适合放在同一综述框架中；
- 用户给的是综述文章 + 若干研究论文：以研究论文为主线，综述文章只作背景补充；
- 数据不可比：明确标注“当前不能做严格横比”。

## Final instruction
你输出的重点不是“谁做过什么”，而是回答：
- 这个方向到底在解决什么；
- 学界已经想通了什么；
- 还有什么没想通；
- 下一步真正值得追的问题在哪。
